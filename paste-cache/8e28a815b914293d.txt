## 4. Extension architecture (updated)

### Manifest (v3)

- `content_scripts` on `https://www.perplexity.ai/discover*`.  
- Permissions:  
  - `"scripting"`, `"storage"`. [developer.chrome](https://developer.chrome.com/docs/extensions/get-started/tutorial/scripts-on-every-tab)
  - Host permissions for `https://www.perplexity.ai/*`.  
- `background.service_worker`:  
  - Handles cross-origin fetch to the story URL (still same origin, but good separation of concerns).  

### Content script

On `https://www.perplexity.ai/discover*`:

1. On load:  
   - Query DOM for all cards using your chosen selector (e.g., `.FeedItem`, `article`, or whatever you find). [developer.chrome](https://developer.chrome.com/docs/extensions/get-started/tutorial/scripts-on-every-tab)
2. For each card:  
   - Extract the story `href`.  
   - Skip if `data-px-date="inserted"` is already set.  
   - Send a message to the background worker `{ type: "GET_PUBLISH_DATE", url }`.  
3. On response `{ dateString }`:  
   - If `dateString` is non-null:  
     - Create a `<span class="px-date-label">Jan 20, 2026</span>`.  
     - Insert it into a consistent spot in the card (e.g., next to the source line).  
     - Mark the card as processed: `card.dataset.pxDate = "inserted"`.  
4. Attach a MutationObserver to the feed container to process new cards in batches (debounced ~500 ms). [developer.mozilla](https://developer.mozilla.org/en-US/docs/Web/API/MutationObserver)

### Background/service worker

- On message `{ type: "GET_PUBLISH_DATE", url }`:  
  1. Check `chrome.storage.local` for a cached value under the URL key.  
  2. If cached, return it.  
  3. Else:  
     - `fetch(url)`.  
     - `const html = await response.text();`  
     - Parse with `DOMParser`: `new DOMParser().parseFromString(html, "text/html")`.  
     - `doc.querySelector(STORY_TIME_SELECTOR)` to find `<time>`.  
     - Read `dateTimeAttr = timeEl.getAttribute("datetime")` or textContent.  
     - Normalize to ISO and to a display format (e.g., `Jan 20, 2026`).  
     - Cache and return.

## 5. Date extraction strategy (simplified)

Instead of scanning arbitrary meta tags across external news sites, you now:

1. Parse the story page HTML into a DOM in the background script.  
2. Select the `<time>` element via the known selector.  
3. Resolve the date:  
   - Prefer `timeEl.getAttribute("datetime")` if present (likely ISO).  
   - Fallback to `timeEl.textContent` and `Date.parse`.  
4. Format with `toLocaleDateString` or a small formatter for `MMM DD, YYYY`.  

You can still keep the more general meta/JSON‑LD logic as a fallback, but in practice Perplexity’s `<time>` should be enough for this use case. [help.leadboxer](https://help.leadboxer.com/leadboxer/integrations/website/track-meta-data-using-json-ld)

## 7. Performance and rate-limits

- Debounce the MutationObserver callback so you handle bursts of cards every ~500 ms.
