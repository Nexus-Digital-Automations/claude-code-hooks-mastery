 Plan: Integrated File Export for WRDS MCP Tools

 Summary

 Add optional export_path and export_format parameters to all 11 existing tools so users can save results directly to files as part of any query.

 Design

 Instead of a separate export tool, each tool gains:
 stock_data(action="daily_prices", permnos=[14593], start_date="2024-01-01",
            end_date="2024-12-31",
            export_path="/path/to/prices.csv")  # ← Optional: saves to file

 When export_path is provided:
 1. Tool executes normally and gets results
 2. Results are saved to the specified file (format from extension)
 3. Response includes both data AND export confirmation

 New Parameters (added to each tool)

 export_path: Annotated[
     str | None,
     Field(description="File path to export results. Format from extension: .csv, .xlsx, .json, .parquet"),
 ] = None

 export_format: Annotated[
     Literal["csv", "xlsx", "json", "parquet"] | None,
     Field(description="Override format (default: detect from file extension)"),
 ] = None

 Shared Utility Module: export_utils.py

 def save_results(
     data: str | list | dict | pd.DataFrame,
     export_path: str,
     export_format: str | None = None,
 ) -> dict:
     """Save data to file, return export info."""

 Usage Examples

 # Stock data → CSV
 stock_data(action="daily_prices", permnos=[14593],
            start_date="2024-01-01", end_date="2024-12-31",
            export_path="/Users/me/prices.csv")

 # Screening → Excel
 screener(as_of_date="2024-12-31", max_pe=15, min_roe=0.10,
          export_path="/Users/me/value_stocks.xlsx")

 # SQL query → Parquet
 wrds_explorer(action="query", sql="SELECT * FROM crsp.dsf LIMIT 1000",
               export_path="/Users/me/query_result.parquet")

 # Analytics → JSON
 analytics(action="factor_exposures", permnos=[14593, 10107],
           as_of_date="2024-12-31", export_path="/Users/me/factors.json")

 Return Value (when export_path provided)

 {
     "data": [...normal tool response...],
     "export": {
         "success": true,
         "file_path": "/absolute/path/to/file.csv",
         "format": "csv",
         "file_size_bytes": 12345,
         "file_size_human": "12.1 KB",
         "rows": 100,
         "columns": 5
     }
 }

 Implementation Steps

 1. Add openpyxl>=3.1.0 to pyproject.toml
 2. Create mcp_server/tools/export_utils.py with shared functions
 3. Modify each of 11 tools to:
   - Import export_utils
   - Add export_path and export_format parameters
   - Call save_results() before returning if export_path provided
   - Wrap response with export info

 Verification

 1. Test each tool with export_path parameter
 2. Verify all 4 formats work (CSV, Excel, JSON, Parquet)
 3. Verify format detection from file extension
 4. Test path validation and directory creation
 5. Verify response includes both data and export info
